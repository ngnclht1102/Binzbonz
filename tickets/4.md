# Task 4 — TypeORM Entities (All Tables)

## Type: Backend

## Description
Create all TypeORM entity classes matching the database schema. Entities are co-located with their modules.

## Entities to Create

### `apps/api/src/actors/actor.entity.ts`
- `id` uuid PK (generated)
- `name` text unique
- `type` text — `human` | `agent`
- `role` text nullable — `developer` | `ctbaceo` | null
- `avatar_url` text nullable
- `session_id` text nullable
- `last_token_count` int default 0
- `last_active_at` timestamptz nullable
- `status` text default `'idle'` — `idle` | `working` | `compacting`
- `created_at` timestamptz (CreateDateColumn)

### `apps/api/src/projects/project.entity.ts`
- `id` uuid PK
- `name` text
- `description` text nullable
- `brief` text nullable
- `repo_path` text nullable
- `main_branch` text default `'main'`
- `worktree_path` text nullable
- `claude_md_path` text nullable
- `status` text default `'analysing'` — `analysing` | `paused` | `active` | `completed`
- `created_at`, `updated_at` (CreateDateColumn, UpdateDateColumn)

### `apps/api/src/hierarchy/mvp.entity.ts`
- `id`, `project` ManyToOne→Project, `title`, `description`, `status`, `created_at`

### `apps/api/src/hierarchy/sprint.entity.ts`
- `id`, `mvp` ManyToOne→Mvp, `title`, `goal`, `start_date`, `end_date`, `status`, `created_at`

### `apps/api/src/hierarchy/epic.entity.ts`
- `id`, `sprint` ManyToOne→Sprint, `title`, `description`, `created_at`

### `apps/api/src/hierarchy/feature.entity.ts`
- `id`, `epic` ManyToOne→Epic, `title`, `description`, `acceptance_criteria` text nullable, `created_at`

### `apps/api/src/tasks/task.entity.ts`
- `id` uuid PK
- `feature` ManyToOne→Feature
- `parent_task` ManyToOne→Task (self-ref, nullable) — for subtasks
- `subtasks` OneToMany→Task
- `title`, `description`
- `status` default `'backlog'` — `backlog` | `assigned` | `in_progress` | `blocked` | `review_request` | `done`
- `assigned_agent` ManyToOne→Actor nullable
- `branch_name` text nullable
- `worktree_path` text nullable
- `worktree_created_at` timestamptz nullable
- `created_by` ManyToOne→Actor nullable
- `priority` int default 0
- `created_at`, `updated_at`

### `apps/api/src/comments/comment.entity.ts`
- `id` uuid PK
- `task` ManyToOne→Task
- `actor` ManyToOne→Actor
- `body` text
- `comment_type` text default `'update'` — `update` | `block` | `question` | `review_request` | `handoff` | `memory_update`
- `created_at`

### `apps/api/src/comments/project-comment.entity.ts`
- `id`, `project` ManyToOne→Project, `actor` ManyToOne→Actor, `body`, `comment_type`, `created_at`

### `apps/api/src/wake-events/wake-event.entity.ts`
- `id` uuid PK
- `agent` ManyToOne→Actor
- `project` ManyToOne→Project
- `triggered_by` text — `mention` | `heartbeat` | `project_created` | `project_resumed`
- `comment` ManyToOne→Comment nullable
- `status` default `'pending'` — `pending` | `processing` | `done` | `failed` | `skipped`
- `created_at`

### `apps/api/src/memory/memory-file.entity.ts`
- `id` uuid PK
- `project` ManyToOne→Project
- `file_path` text
- `last_updated_at` timestamptz
- `last_updated_by` ManyToOne→Actor
- `git_commit` text nullable

## Notes
- Use `@PrimaryGeneratedColumn('uuid')` for all IDs
- Use `@CreateDateColumn()` and `@UpdateDateColumn()` where applicable
- Register each entity in its module via `TypeOrmModule.forFeature([Entity])`
- Create stub modules for each domain (actors, projects, hierarchy, tasks, comments, wake-events, memory) — just module + entity registration, no controllers/services yet

## Acceptance Criteria
- [ ] `pnpm dev --filter api` boots and TypeORM `synchronize: true` creates all tables
- [ ] Connect to embedded Postgres and verify all tables exist with correct columns
- [ ] Foreign key relationships are correct (e.g. task → feature, comment → task)
- [ ] Self-referencing task → parent_task works
