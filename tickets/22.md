# Task 22 — QA: Frontend + Backend Realtime & Comments

## Type: QA (Manual)

## Prerequisites
Both apps running. A project with tasks exists from QA 20.

## Verify

### Agent Pool
1. Navigate to `/projects/:id/agents`
   - [ ] All 7 agents displayed (1 ctbaceo + 6 developers)
   - [ ] Each shows name, role badge, status dot
   - [ ] CTBACEO has a distinct role badge from developers

2. Create a new agent via "Create Agent" button
   - [ ] Dialog opens with name input
   - [ ] Submit → new agent appears in the grid
   - [ ] `GET /actors` confirms the agent exists in the API

### Comments + Mentions
3. Open a task in the board view
   - [ ] Comment input visible at bottom of detail panel
   - [ ] Select `comment_type` from dropdown (default: `update`)

4. Post a comment: "Hey @dev-1 please review this"
   - [ ] Comment appears in the list immediately
   - [ ] `@dev-1` is visually highlighted in the comment body
   - [ ] Actor name and timestamp shown

5. Post a comment mentioning `@ctbaceo`
   - [ ] Comment posted successfully
   - [ ] Wake event created (verify via `GET /wake-events?status=pending`)

### Realtime (SSE)
6. Open the project board view in **Browser A**

7. In a terminal, create a comment via curl:
   ```bash
   curl -X POST http://localhost:3001/tasks/<taskId>/comments \
     -H 'Content-Type: application/json' \
     -d '{"actor_id":"<devId>","body":"External comment","comment_type":"update"}'
   ```
   - [ ] Browser A shows the new comment WITHOUT manual refresh

8. In a terminal, update a task status:
   ```bash
   curl -X PATCH http://localhost:3001/tasks/<taskId> \
     -H 'Content-Type: application/json' \
     -d '{"status":"in_progress"}'
   ```
   - [ ] Browser A board view moves the task card to the correct column WITHOUT refresh

9. **Reconnection** — in Browser A DevTools, go offline then back online
   - [ ] SSE reconnects (check Network tab for new EventSource connection)
   - [ ] Subsequent changes still update in realtime

### Human Identity
10. Check localStorage in browser DevTools
    - [ ] `username` key exists (or is prompted on first use)
    - [ ] Comments posted from UI use the correct actor_id matching the stored name

## Pass Criteria
Realtime updates work across API mutations and UI. Comments with mentions create wake events. Agent pool CRUD works. If any fail, reopen Task 21.
