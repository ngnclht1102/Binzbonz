# Task 8 — Projects CRUD Module

## Type: Backend

## Description
Build the Projects NestJS module with full CRUD REST API.

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/projects` | List all projects |
| GET | `/projects/:id` | Get project by ID (include hierarchy counts) |
| POST | `/projects` | Create a new project (name + brief) |
| PATCH | `/projects/:id` | Update project fields (including status transitions) |
| DELETE | `/projects/:id` | Delete project |

## Files
- `apps/api/src/projects/projects.module.ts`
- `apps/api/src/projects/projects.controller.ts`
- `apps/api/src/projects/projects.service.ts`
- `apps/api/src/projects/dto/create-project.dto.ts`
- `apps/api/src/projects/dto/update-project.dto.ts`

## Notes
- `POST /projects` only requires `name` and `brief` — status defaults to `analysing`
- `PATCH /projects/:id` can update `status` — validate transitions:
  - `analysing` → `paused` | `active`
  - `paused` → `analysing` | `active`
  - `active` → `paused` | `completed`
  - `completed` → (no transitions)
- `GET /projects/:id` should return nested counts: `{ mvpCount, taskCount, ... }`

## Acceptance Criteria
- [ ] `POST /projects` with `{ name, brief }` creates project in `analysing` status
- [ ] `GET /projects` lists all projects
- [ ] `GET /projects/:id` returns project with hierarchy counts
- [ ] `PATCH /projects/:id { status: 'active' }` transitions correctly
- [ ] Invalid status transitions return 400
- [ ] `DELETE /projects/:id` removes the project
