# Task 13 — QA: All REST API Endpoints

## Type: QA (Manual)

## Verify
Test every endpoint using `curl` or a REST client (Postman, Insomnia, HTTPie). The API should be running at `http://localhost:3001`.

### 1. Actors
- [ ] `GET /actors` → returns 8 seeded actors
- [ ] `GET /actors?type=agent&role=developer` → returns 6 developers
- [ ] `POST /actors { "name": "test-agent", "type": "agent", "role": "developer" }` → 201
- [ ] `PATCH /actors/:id { "status": "working" }` → updated
- [ ] `DELETE /actors/:id` → removed
- [ ] `GET /actors/nonexistent-uuid` → 404

### 2. Projects
- [ ] `POST /projects { "name": "Test Project", "brief": "Build something" }` → created in `analysing` status
- [ ] `GET /projects` → lists the project
- [ ] `PATCH /projects/:id { "status": "active" }` → transitions to active
- [ ] `PATCH /projects/:id { "status": "completed" }` from `analysing` → 400 (invalid transition)

### 3. Hierarchy
- [ ] Create MVP under project → 201
- [ ] Create Sprint under MVP → 201
- [ ] Create Epic under Sprint → 201
- [ ] Create Feature under Epic → 201
- [ ] Create Feature under nonexistent Epic → 404
- [ ] Delete MVP → cascades and removes sprint/epic/feature

### 4. Tasks
- [ ] Create task under feature → 201, status = `backlog`
- [ ] Create subtask under task → 201, has `parent_task_id`
- [ ] Assign agent → status auto-changes to `assigned`
- [ ] `GET /tasks/:id` includes subtasks array
- [ ] `GET /projects/:projectId/tasks` returns tasks across hierarchy

### 5. Comments + Mentions
- [ ] `POST /tasks/:taskId/comments { "actor_id": "...", "body": "Hey @dev-1 check this", "comment_type": "update" }` → 201
- [ ] Verify a `wake_event` was created for `dev-1` with `triggered_by: 'mention'`
- [ ] Comment mentioning `@brian` → no wake event (human)
- [ ] `GET /tasks/:taskId/comments` → ordered by created_at

### 6. Wake Events
- [ ] `GET /wake-events?status=pending` → shows the mention-created event
- [ ] `PATCH /wake-events/:id { "status": "done" }` → updated

### 7. Memory Files
- [ ] `POST /projects/:projectId/memory-files { ... }` → created
- [ ] `GET /projects/:projectId/memory-files/changed?since=...` → correct filtering

## Pass Criteria
All endpoints return expected status codes and payloads. If any fail, reopen the relevant task (7-12).
