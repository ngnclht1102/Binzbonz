# Task 19 — Frontend: Board View + Tree View

## Type: Frontend

## Description
Build the Board view (kanban-style task columns) and Tree view (hierarchy visualization) for the project detail page.

## Board View (`/projects/:id`)

### Layout
- Columns by task status: `backlog` | `assigned` | `in_progress` | `blocked` | `review_request` | `done`
- Each column shows task cards
- Fetch `GET /projects/:projectId/tasks` on mount

### Task Card
- Title, status badge, assigned agent name (or "Unassigned")
- Priority indicator
- Subtask count (if any)
- Click → expand to show description, comments count, branch name

### Interactions
- Click task card → open task detail panel (slide-over or modal)
- Task detail shows: full description, assigned agent, status, branch, subtasks list, comments list
- Comment input at bottom of task detail → `POST /tasks/:taskId/comments`

## Tree View (`/projects/:id/tree`)

### Layout
- Hierarchical tree: MVP → Sprint → Epic → Feature → Task → Subtask
- Fetch full hierarchy: `GET /projects/:projectId/mvps` then nested calls, or a single endpoint if available
- Collapsible tree nodes
- Each node shows: title, type badge, status

### Interactions
- Click any node → show details in side panel
- Expand/collapse nodes

## Notes
- Use Zustand `tasks-store` for task data
- Board columns should show count in header (e.g. "In Progress (3)")
- Tree view can use recursive component rendering
- Both views share the same project detail layout (tabs)

## Acceptance Criteria
- [ ] Board view shows task cards organized by status columns
- [ ] Task cards display title, agent, status badge
- [ ] Clicking a task opens detail panel with description and comments
- [ ] Can post a comment from the task detail panel
- [ ] Tree view renders the full hierarchy as a collapsible tree
- [ ] Tree nodes are expandable/collapsible
- [ ] Both views load data from the API
