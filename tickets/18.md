# Task 18 — QA: Frontend + Backend Projects Integration

## Type: QA (Manual)

## Prerequisites
Both `apps/api` and `apps/web` must be running (`pnpm dev`).

## Verify

1. **Project list** — navigate to `http://localhost:3000/projects`
   - [ ] Page loads without errors
   - [ ] Seeded projects (if any) or empty state displayed
   - [ ] "New Project" button visible

2. **Create project** — click "New Project", fill form:
   - Name: "Test Orchestration Project"
   - Brief: "A platform for managing AI agents"
   - [ ] Submit → redirected to `/projects/:id`
   - [ ] Project detail page shows correct name, brief, status = `analysing`

3. **Project appears in list** — go back to `/projects`
   - [ ] New project appears in the card grid
   - [ ] Status badge shows "analysing" in yellow

4. **Navigation** — click into the project
   - [ ] Tab navigation shows Board | Tree | Agents
   - [ ] Clicking tabs changes URL (`/projects/:id`, `/projects/:id/tree`, `/projects/:id/agents`)
   - [ ] No console errors in browser DevTools

5. **Error handling** — try creating a project with empty name
   - [ ] Validation prevents submission or shows error

6. **API connectivity** — open browser DevTools Network tab
   - [ ] Requests go to `localhost:3001` and return 200
   - [ ] No CORS errors

## Pass Criteria
Full create → view → list flow works end-to-end. If CORS issues, add CORS config to NestJS `main.ts`. If any fail, reopen Task 16 or 17.
