# Task 5 — Seed Data Module

## Type: Backend

## Description
Create a `SeedModule` that runs on application startup and idempotently seeds the database with default actors.

## Requirements

### `apps/api/src/seed/seed.module.ts` + `seed.service.ts`
- Implements `OnModuleInit` → runs `seed()` on startup
- Upserts (insert-if-not-exists by `name`):
  - 1 CTBACEO agent: `{ name: 'ctbaceo', type: 'agent', role: 'ctbaceo', status: 'idle' }`
  - 6 developer agents: `dev-1` through `dev-6`, `{ type: 'agent', role: 'developer', status: 'idle' }`
  - 1 human actor: `{ name: 'brian', type: 'human', role: null, status: 'idle' }`
- Use TypeORM `upsert` or `INSERT ... ON CONFLICT (name) DO NOTHING`
- Log how many actors were seeded vs already existed

## Acceptance Criteria
- [ ] First run creates 8 actors in the `actor` table
- [ ] Second run logs "already seeded" or similar — no duplicates
- [ ] `SELECT * FROM actor` returns all 8 rows with correct types and roles
