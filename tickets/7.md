# Task 7 — Actors CRUD Module

## Type: Backend

## Description
Build the Actors NestJS module with full CRUD REST API.

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/actors` | List all actors (optional `?type=agent&role=developer` filters) |
| GET | `/actors/:id` | Get actor by ID |
| POST | `/actors` | Create a new actor |
| PATCH | `/actors/:id` | Update actor fields |
| DELETE | `/actors/:id` | Delete actor |

## Files
- `apps/api/src/actors/actors.module.ts`
- `apps/api/src/actors/actors.controller.ts`
- `apps/api/src/actors/actors.service.ts`
- `apps/api/src/actors/dto/create-actor.dto.ts`
- `apps/api/src/actors/dto/update-actor.dto.ts`

## Notes
- Use `class-validator` decorators on DTOs
- `ValidationPipe` should be enabled globally in `main.ts`
- GET `/actors` supports query params: `type`, `role`, `status`
- Return 404 if actor not found on GET/PATCH/DELETE

## Acceptance Criteria
- [ ] `POST /actors` creates an actor and returns it with generated UUID
- [ ] `GET /actors` returns all actors (including seeded ones)
- [ ] `GET /actors?type=agent&role=developer` returns only developer agents
- [ ] `PATCH /actors/:id` updates fields and returns updated entity
- [ ] `DELETE /actors/:id` removes the actor
- [ ] Invalid payloads return 400 with validation errors
