# Task 25 — QA: End-to-End Flow

## Type: QA (Manual)

## Description
Full end-to-end test of the entire platform: create a project, build hierarchy, assign agents, verify comments and realtime, test agent runner.

## Prerequisites
- All three apps running: `pnpm dev` (api, web, agent-runner)
- Claude CLI installed and available on PATH
- A git repo configured for worktree testing

## Test Flow

### 1. Create Project via UI
- [ ] Go to `http://localhost:3000/projects/new`
- [ ] Create project: name = "E2E Test", brief = "Build a todo app"
- [ ] Redirected to project detail, status = `analysing`

### 2. Build Hierarchy via API
```bash
# Create MVP, Sprint, Epic, Feature, Tasks
# (use curl commands from QA 20 setup, targeting the new project)
```
- [ ] Hierarchy created successfully
- [ ] Tree view (`/projects/:id/tree`) shows full hierarchy

### 3. Assign Agent to Task
```bash
curl -X PATCH http://localhost:3001/tasks/<taskId> \
  -H 'Content-Type: application/json' \
  -d '{"assigned_agent_id":"<dev-1-id>"}'
```
- [ ] Task status auto-changes to `assigned`
- [ ] Board view shows task in "Assigned" column
- [ ] Agent pool shows dev-1 status update

### 4. Trigger Agent via Mention
- [ ] Post comment on task via UI: "Hey @dev-1 start working on this"
- [ ] `wake_event` created (check `GET /wake-events?status=pending`)
- [ ] Agent runner picks up the event (check runner logs)
- [ ] Agent runner spawns Claude CLI (or logs attempt if Claude not configured)

### 5. Verify Agent Runner Behaviour
- [ ] Wake event transitions: `pending` → `processing` → `done` (or `failed`)
- [ ] Actor `last_active_at` and `last_token_count` updated
- [ ] If Claude responded, response posted as comment on the task
- [ ] Comment appears in UI via SSE (realtime)

### 6. Git Worktree (if repo configured)
- [ ] Transition task to `in_progress`
- [ ] Worktree created at configured path
- [ ] `memory/` symlinked inside worktree
- [ ] `git worktree list` shows the new worktree
- [ ] Transition task to `done` → worktree removed

### 7. Project Status Transitions
- [ ] Transition project to `active` — all agents can work
- [ ] Transition project to `paused` — only CTBACEO events are processed
- [ ] Non-CTBACEO wake events during pause → marked `skipped`

### 8. Session Management
- [ ] After agent run, `session_id` stored on actor
- [ ] Next wake for same agent uses `--resume <session_id>`
- [ ] Token count tracked and updated

### 9. Stress Test
- [ ] Create 3 tasks, assign different agents, mention all 3
- [ ] Agent runner processes events sequentially
- [ ] All 3 wake events reach `done` status
- [ ] Board view updates in realtime for all changes

## Pass Criteria
The full lifecycle works: create project → build hierarchy → assign agents → trigger via mention → agent runs → output posted as comments → UI updates in realtime. If any component fails, reopen the relevant task.
