# Task 3 — Embedded Postgres + DatabaseModule

## Type: Backend

## Description
Create a NestJS `DatabaseModule` that boots an embedded Postgres instance on startup using `embedded-postgres@^18.1.0-beta.16`, then exposes the connection to TypeORM.

## Requirements

### Dependencies
```
pnpm --filter api add embedded-postgres@^18.1.0-beta.16 @nestjs/typeorm typeorm pg
```

### DatabaseModule (`apps/api/src/database/`)
- `database.module.ts` — global module, imports `TypeOrmModule.forRootAsync()`
- `embedded-postgres.service.ts` — handles embedded Postgres lifecycle:
  1. Dynamically import `embedded-postgres`
  2. If `DATABASE_URL` env var is set → skip embedded, return external URL
  3. Check for existing cluster via `PG_VERSION` file in `apps/api/data/postgres/`
  4. Check `postmaster.pid` — if running process exists, reuse it
  5. Remove stale `postmaster.pid` if process is dead
  6. Auto-detect next free port if `54329` is busy (use `detect-port` or similar)
  7. `instance.initialise()` on first run (`--encoding=UTF8 --locale=C`)
  8. `instance.start()` to boot Postgres
  9. `ensureDatabase()` — create `binzbonz` database via raw SQL if not exists
  10. Return connection string `postgres://binzbonz:binzbonz@127.0.0.1:<port>/binzbonz`
- Register `SIGINT`/`SIGTERM` handlers to call `instance.stop()`
- Constructor options:
  ```typescript
  new EmbeddedPostgres({
    databaseDir: dataDir,
    user: "binzbonz",
    password: "binzbonz",
    port: selectedPort,
    persistent: true,
    initdbFlags: ["--encoding=UTF8", "--locale=C"],
    onLog: (msg) => logger.debug(msg),
    onError: (msg) => logger.error(msg),
  });
  ```

### TypeORM Integration
- `TypeOrmModule.forRootAsync()` injects `EmbeddedPostgresService` to get connection URL
- Config: `synchronize: true`, `autoLoadEntities: true`, `type: 'postgres'`

### Gitignore
- Add `apps/api/data/` to `.gitignore`

## Acceptance Criteria
- [ ] `pnpm dev --filter api` boots embedded Postgres and logs connection info
- [ ] First run creates `apps/api/data/postgres/` with `PG_VERSION` file
- [ ] Second run reuses existing cluster (no re-init)
- [ ] Setting `DATABASE_URL=postgres://...` skips embedded and connects externally
- [ ] `Ctrl+C` cleanly stops embedded Postgres (no orphan process)
