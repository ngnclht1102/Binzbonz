# Task 15 — QA: SSE Realtime

## Type: QA (Manual)

## Verify

1. **Connect to SSE** — open a terminal and run:
   ```bash
   curl -N http://localhost:3001/events/stream
   ```
   - [ ] Connection stays open, no immediate close

2. **Comment trigger** — in another terminal, create a comment:
   ```bash
   curl -X POST http://localhost:3001/tasks/<taskId>/comments \
     -H 'Content-Type: application/json' \
     -d '{"actor_id":"<actorId>","body":"test","comment_type":"update"}'
   ```
   - [ ] SSE terminal shows `event: comment_change` with correct task_id

3. **Task trigger** — update a task:
   ```bash
   curl -X PATCH http://localhost:3001/tasks/<taskId> \
     -H 'Content-Type: application/json' \
     -d '{"status":"in_progress"}'
   ```
   - [ ] SSE terminal shows `event: task_change` with correct id

4. **Wake event trigger** — the comment with `@dev-1` from QA task 13 should also trigger:
   - [ ] SSE terminal shows `event: wake_event_change`

5. **Multiple clients** — open two `curl -N` connections:
   - [ ] Both receive the same events when data changes

6. **Reconnection** — disconnect and reconnect SSE:
   - [ ] New connection works, receives subsequent events

## Pass Criteria
All SSE events fire correctly for data mutations. If any fail, reopen Task 14.
