# Task 16 — Frontend Scaffolding (Layout, Routing, Zustand, API Client)

## Type: Frontend

## Description
Set up the Next.js frontend with app router layout, routing structure, Zustand stores, Tailwind theme, shadcn/ui components, and a typed API client.

## Requirements

### Layout (`apps/web/app/layout.tsx`)
- Root layout with Tailwind dark theme (dark bg, light text)
- Sidebar navigation (collapsible) with links to `/projects`
- Top bar with app name "Binzbonz"

### Routing
```
app/
├── layout.tsx              # root layout with sidebar
├── page.tsx                # redirect to /projects
├── projects/
│   ├── page.tsx            # project list
│   ├── new/
│   │   └── page.tsx        # create project form
│   └── [id]/
│       ├── page.tsx        # board view (default)
│       ├── tree/
│       │   └── page.tsx    # tree view
│       └── agents/
│           └── page.tsx    # agent pool
```
- All pages can be placeholder/skeleton for now — just routing must work

### API Client (`apps/web/lib/api.ts`)
- Base URL: `http://localhost:3001` (configurable via `NEXT_PUBLIC_API_URL`)
- Typed fetch wrapper for all endpoints (actors, projects, hierarchy, tasks, comments, wake-events)
- Returns typed responses matching backend DTOs

### Zustand Stores (`apps/web/lib/stores/`)
- `projects-store.ts` — project list + active project
- `tasks-store.ts` — tasks for active project
- `actors-store.ts` — actor list
- `events-store.ts` — SSE connection manager, dispatches to other stores on events

### shadcn/ui Components
- Init shadcn/ui if not done in Task 1
- Add commonly needed components: Button, Card, Input, Textarea, Badge, Dialog, Select, Tabs

## Acceptance Criteria
- [ ] `pnpm dev --filter web` starts Next.js on port 3000
- [ ] `/` redirects to `/projects`
- [ ] `/projects`, `/projects/new`, `/projects/:id`, `/projects/:id/tree`, `/projects/:id/agents` all render placeholder pages
- [ ] Sidebar navigation links work
- [ ] API client can fetch `GET /actors` from backend (with both apps running)
