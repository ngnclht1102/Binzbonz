# Task 17 — Frontend: Projects Pages (List, Create, Detail)

## Type: Frontend

## Description
Build the project list, create project form, and project detail shell pages.

## Pages

### `/projects` — Project List
- Fetch `GET /projects` on mount
- Display as card grid: project name, status badge, brief (truncated), created date
- Status badges: `analysing` (yellow), `paused` (gray), `active` (green), `completed` (blue)
- Click card → navigate to `/projects/:id`
- "New Project" button → navigate to `/projects/new`

### `/projects/new` — Create Project
- Form with fields: `name` (required), `brief` (required, textarea)
- Submit → `POST /projects` → redirect to `/projects/:id`
- Cancel → back to `/projects`
- Validation: name must not be empty, brief must not be empty

### `/projects/:id` — Project Detail (Board View Shell)
- Fetch `GET /projects/:id` on mount
- Header: project name, status badge, brief (expandable)
- Tab navigation: Board (default) | Tree | Agents
- Board view content built in Task 19
- For now: show project info + placeholder for board

## Notes
- Use Zustand `projects-store` to cache project list
- Use shadcn/ui Card, Badge, Input, Textarea, Button components
- Handle loading states and error states
- API errors should show toast or inline error message

## Acceptance Criteria
- [ ] `/projects` shows all projects from the API as cards
- [ ] Clicking a card navigates to `/projects/:id`
- [ ] `/projects/new` form creates a project and redirects to detail
- [ ] `/projects/:id` displays project info with tab navigation
- [ ] Status badges render with correct colors
