# Task 24 — Git Worktree Management + Skill Files

## Type: Backend

## Description
Add git worktree lifecycle management to the task workflow and create the CLAUDE.md + skill files for agents.

## Git Worktree Management

### Worktree Service (`apps/api/src/tasks/worktree.service.ts`)
Uses `execa` to run git commands.

**Create worktree** — called when a developer agent starts a task (status → `in_progress`):
```bash
git fetch origin
git worktree add <project.worktree_path>/task-<taskId> -b task/<taskId> origin/main
ln -s <project.repo_path>/memory <project.worktree_path>/task-<taskId>/memory
```
- Update task entity: `branch_name`, `worktree_path`, `worktree_created_at`

**Remove worktree** — called when task status → `done`:
```bash
git worktree remove <task.worktree_path>
git branch -d task/<taskId>
```
- Clear task entity: `worktree_path`, `worktree_created_at`

### Integration with Tasks Module
- Hook into task status changes in `TasksService`:
  - `assigned` → `in_progress`: call `createWorktree()`
  - Any status → `done`: call `removeWorktree()`
- Worktree operations are best-effort — log errors but don't block status change

## Skill Files

### `skills/developer.md`
Agent skill file for developer agents. Contents should cover:
- Role: fullstack developer
- Workflow: read task, create worktree, implement, self-QA, write tests, post update comments
- Definition of done: tests pass, no lint errors, code reviewed
- Memory: read `memory/` directory for project context
- Communication: post comments with appropriate `comment_type`
- Git: work in assigned worktree, commit frequently

### `skills/ctbaceo.md`
Agent skill file for CTBACEO agent. Contents should cover:
- Role: project coordinator and technical lead
- Workflow: analyse brief → create hierarchy → assign tasks → monitor progress
- Heartbeat: scan all tasks, ping stuck/idle agents
- Memory guardianship: review `memory_update` comments, apply changes
- QA: review completed tasks, approve or request changes
- Agent creation: can create new developer agents when pool is exhausted

### `CLAUDE.md` (project root)
- Points to skill files: "If you are a developer agent, read `skills/developer.md`. If you are CTBACEO, read `skills/ctbaceo.md`."
- Points to `memory/` directory for project context
- Basic repo structure overview

## Acceptance Criteria
- [ ] Transitioning a task to `in_progress` creates a git worktree at the configured path
- [ ] Worktree is branched from `origin/main`
- [ ] `memory/` symlink exists inside the worktree
- [ ] Task entity updated with `branch_name`, `worktree_path`, `worktree_created_at`
- [ ] Transitioning a task to `done` removes the worktree and branch
- [ ] `skills/developer.md` and `skills/ctbaceo.md` exist with meaningful content
- [ ] `CLAUDE.md` exists at project root
