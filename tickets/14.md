# Task 14 — SSE Realtime Module (pg_notify + SSE)

## Type: Backend

## Description
Implement Postgres-based realtime notifications using `pg_notify` triggers and a NestJS SSE endpoint.

## Requirements

### Postgres Triggers
Create raw SQL (run via TypeORM query runner on module init) to add triggers on:
- `comment` table → `NOTIFY 'comment_change', '{"id":"...","task_id":"...","action":"INSERT"}'`
- `task` table → `NOTIFY 'task_change', '{"id":"...","action":"INSERT|UPDATE"}'`
- `wake_event` table → `NOTIFY 'wake_event_change', '{"id":"...","action":"INSERT|UPDATE"}'`

Triggers fire on INSERT and UPDATE using `TG_OP` to determine action.

### SSE Controller
- `GET /events/stream` — SSE endpoint
- Use a raw `pg` client (separate from TypeORM pool) to `LISTEN` on all three channels
- On notification, push to all connected SSE clients
- Event format:
  ```
  event: comment_change
  data: {"id":"uuid","task_id":"uuid","action":"INSERT"}

  event: task_change
  data: {"id":"uuid","action":"UPDATE"}
  ```

### Files
- `apps/api/src/events/events.module.ts`
- `apps/api/src/events/events.controller.ts` (SSE endpoint)
- `apps/api/src/events/events.gateway.ts` (pg LISTEN manager)
- `apps/api/src/events/pg-notify.setup.ts` (trigger SQL)

## Notes
- Use `@Sse()` decorator from `@nestjs/common`
- Return `Observable<MessageEvent>` from the SSE handler
- The pg LISTEN client must be a separate connection — not from the TypeORM pool
- Triggers should use `CREATE OR REPLACE FUNCTION` + `DROP TRIGGER IF EXISTS` for idempotency

## Acceptance Criteria
- [ ] `GET /events/stream` returns `Content-Type: text/event-stream`
- [ ] Creating a comment via API → SSE client receives `comment_change` event
- [ ] Updating a task status → SSE client receives `task_change` event
- [ ] Creating a wake event → SSE client receives `wake_event_change` event
- [ ] Multiple SSE clients receive the same events
