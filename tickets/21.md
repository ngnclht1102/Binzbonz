# Task 21 — Frontend: Agent Pool + Comments UI + Realtime (EventSource)

## Type: Frontend

## Description
Build the agent pool page, polish the comments UI, and wire up SSE for realtime updates across the frontend.

## Agent Pool Page (`/projects/:id/agents`)

### Layout
- Grid of agent cards
- Fetch `GET /actors?type=agent` on mount
- Each card shows: name, role badge (`ctbaceo` / `developer`), status indicator, last active time

### Status Indicators
- `idle` — gray dot
- `working` — green pulsing dot
- `compacting` — yellow dot

### Interactions
- "Create Agent" button → dialog to create a new developer agent (name input)
- Submit → `POST /actors { type: 'agent', role: 'developer', name: '...' }`

## Comments UI Polish
- In task detail panel (from Task 19), ensure:
  - Comments render with: actor name, avatar placeholder, timestamp, body, comment_type badge
  - Mention `@name` patterns in body are highlighted/bolded
  - Comment input has a `comment_type` selector (dropdown)
  - New comments appear immediately after POST (optimistic or refetch)

## Realtime (SSE)
- `events-store.ts` manages an `EventSource` connection to `GET /events/stream`
- On `comment_change` event → refetch comments for the active task
- On `task_change` event → refetch tasks for the active project (updates board)
- On `wake_event_change` event → refetch agents (updates status indicators)
- Auto-reconnect on connection drop (with exponential backoff)

## Notes
- Use `useEffect` to connect EventSource when entering a project, disconnect on leave
- Actor status updates come via wake_event_change → re-fetch actors to get latest status
- Human identity stored in localStorage (BR-12): `localStorage.getItem('username')` → look up actor by name for posting comments

## Acceptance Criteria
- [ ] Agent pool page shows all agents with correct status indicators
- [ ] Can create a new developer agent from the UI
- [ ] Comments in task detail show actor name, timestamp, type badge
- [ ] @mentions are visually highlighted in comment body
- [ ] SSE connection established on project page load
- [ ] Creating a comment via API (curl) → UI updates without manual refresh
- [ ] Updating a task via API → board view updates in realtime
- [ ] SSE auto-reconnects after disconnect
