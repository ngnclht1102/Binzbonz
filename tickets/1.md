# Task 1 — Monorepo Scaffolding

## Type: Setup

## Description
Scaffold the pnpm monorepo with Turborepo containing three apps: NestJS API, Next.js web frontend, and agent-runner worker.

## Requirements
- Initialise root `package.json` with `"private": true`
- Create `pnpm-workspace.yaml` referencing `apps/*` and `agent-runner`
- Create `turbo.json` with `build`, `dev`, `lint` pipelines
- Scaffold `apps/api` — NestJS app (strict TypeScript)
  - `pnpm create nest` or manual setup with `@nestjs/cli`
  - Ensure `apps/api/src/main.ts` boots on port `3001`
- Scaffold `apps/web` — Next.js 14 app router
  - `pnpm create next-app` with TypeScript, Tailwind CSS, app router
  - Install and init `shadcn/ui`
  - Ensure dev server runs on port `3000`
- Scaffold `agent-runner` — plain TypeScript Node.js app
  - `src/index.ts` entry point, `tsx` for dev, `tsup` for build
- Root scripts:
  - `pnpm dev` → runs all three apps via turbo
  - `pnpm build` → builds all three apps via turbo
- Add root `.gitignore` (node_modules, dist, .next, apps/api/data/)
- Add root `tsconfig.base.json` with shared compiler options

## Acceptance Criteria
- [ ] `pnpm install` succeeds from root with no errors
- [ ] `pnpm dev` starts all three apps concurrently
- [ ] NestJS API responds at `http://localhost:3001`
- [ ] Next.js app loads at `http://localhost:3000`
- [ ] `pnpm build` compiles all three apps without errors
