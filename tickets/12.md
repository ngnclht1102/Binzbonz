# Task 12 — Wake Events + Memory Files Modules

## Type: Backend

## Description
Build the Wake Events and Memory Files NestJS modules with CRUD endpoints.

## Wake Events Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/wake-events` | List wake events (filter by `?status=pending&agent_id=...`) |
| GET | `/wake-events/:id` | Get wake event by ID |
| POST | `/wake-events` | Create wake event manually (for heartbeat/project_created triggers) |
| PATCH | `/wake-events/:id` | Update status (pending → processing → done/failed/skipped) |

## Memory Files Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/projects/:projectId/memory-files` | List memory files for a project |
| POST | `/projects/:projectId/memory-files` | Register a memory file |
| PATCH | `/memory-files/:id` | Update (new commit, updated_by, updated_at) |
| GET | `/projects/:projectId/memory-files/changed?since=<timestamp>` | Get files changed since timestamp |

## Notes
- Wake event status transitions: `pending` → `processing` → `done` | `failed` | `skipped`
- The "changed since" endpoint is used by agent runner for memory sync (§4.7)
- Wake events are the job queue — no Redis needed

## Acceptance Criteria
- [ ] `POST /wake-events` creates event with `status: 'pending'`
- [ ] `GET /wake-events?status=pending` returns only pending events
- [ ] `PATCH /wake-events/:id { status: 'processing' }` works
- [ ] Memory files CRUD works
- [ ] `GET /projects/:id/memory-files/changed?since=2024-01-01` returns only files updated after that date
