# Task 20 — QA: Frontend + Backend Board & Tree Views

## Type: QA (Manual)

## Prerequisites
Both apps running. You need a project with hierarchy data. Create test data first:

```bash
# Create project (if not from QA 18)
PROJECT_ID=$(curl -s -X POST http://localhost:3001/projects \
  -H 'Content-Type: application/json' \
  -d '{"name":"QA Project","brief":"Testing views"}' | jq -r '.id')

# Create hierarchy
MVP_ID=$(curl -s -X POST http://localhost:3001/projects/$PROJECT_ID/mvps \
  -H 'Content-Type: application/json' \
  -d '{"title":"MVP 1","description":"First MVP"}' | jq -r '.id')

SPRINT_ID=$(curl -s -X POST http://localhost:3001/mvps/$MVP_ID/sprints \
  -H 'Content-Type: application/json' \
  -d '{"title":"Sprint 1","goal":"Setup"}' | jq -r '.id')

EPIC_ID=$(curl -s -X POST http://localhost:3001/sprints/$SPRINT_ID/epics \
  -H 'Content-Type: application/json' \
  -d '{"title":"Epic 1","description":"Core setup"}' | jq -r '.id')

FEATURE_ID=$(curl -s -X POST http://localhost:3001/epics/$EPIC_ID/features \
  -H 'Content-Type: application/json' \
  -d '{"title":"Feature 1","description":"Auth flow"}' | jq -r '.id')

# Create tasks in different statuses
curl -s -X POST http://localhost:3001/features/$FEATURE_ID/tasks \
  -H 'Content-Type: application/json' \
  -d '{"title":"Task Backlog","description":"A backlog task"}'

curl -s -X POST http://localhost:3001/features/$FEATURE_ID/tasks \
  -H 'Content-Type: application/json' \
  -d '{"title":"Task In Progress","description":"Doing work"}'
# Then PATCH the second task to in_progress status
```

## Verify

### Board View (`/projects/:id`)
- [ ] Columns render for each status
- [ ] Task cards appear in correct columns
- [ ] Cards show title and assigned agent (or "Unassigned")
- [ ] Column headers show task count
- [ ] Clicking a task opens detail panel
- [ ] Detail panel shows description, status, comments
- [ ] Can type and submit a comment from the detail panel
- [ ] Comment appears in the list after submission

### Tree View (`/projects/:id/tree`)
- [ ] Full hierarchy renders: MVP → Sprint → Epic → Feature → Task
- [ ] Nodes are collapsible (click to expand/collapse)
- [ ] Each node shows title and type badge
- [ ] Clicking a node shows details

### Navigation
- [ ] Switching between Board and Tree tabs preserves project context
- [ ] Browser back/forward works correctly
- [ ] No console errors

## Pass Criteria
Both views render correctly with real API data. If any fail, reopen Task 19.
