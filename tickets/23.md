# Task 23 — Agent Runner Worker Process

## Type: Backend (agent-runner)

## Description
Build the agent-runner worker that polls `wake_event` table and spawns Claude Code sessions for agents.

## Requirements

### Polling Loop (`agent-runner/src/index.ts`)
- Poll `GET /wake-events?status=pending` every 2 seconds
- For each pending event:
  1. `PATCH /wake-events/:id { status: 'processing' }`
  2. Load actor via `GET /actors/:agentId`
  3. Load project via `GET /projects/:projectId`
  4. **Gate check**: if `project.status` is `analysing` or `paused` and `actor.role !== 'ctbaceo'` → skip (mark `skipped`)
  5. Build prompt (see below)
  6. Spawn Claude session
  7. Parse output
  8. Update actor and mark event done

### Prompt Builder (`agent-runner/src/prompt-builder.ts`)
- If `actor.last_token_count > 960000` → inject `/compact` command immediately
- If `actor.last_token_count > 900000` → prepend compact instruction to prompt
- Otherwise build delta prompt:
  - Fetch new comments since `actor.last_active_at` via `GET /tasks/:taskId/comments`
  - Fetch changed memory files via `GET /projects/:projectId/memory-files/changed?since=<last_active_at>`
  - Compose prompt with new context

### Claude Spawner (`agent-runner/src/claude-spawner.ts`)
- Use `execa` to spawn:
  ```
  claude --resume <session_id>
         --dangerously-skip-permissions
         --output-format stream-json
         -p "<prompt>"
  ```
- If no `session_id` exists (first run), omit `--resume` — capture new session ID from output
- Stream parse the JSON output:
  - Extract `text` blocks → post as comments via `POST /tasks/:taskId/comments`
  - Extract `result` block → read `input_tokens` for token count

### Post-Run Update
- `PATCH /actors/:id` with: `session_id`, `last_token_count`, `last_active_at: now()`, `status: 'idle'`
- `PATCH /wake-events/:id { status: 'done' }`
- On error: `PATCH /wake-events/:id { status: 'failed' }`

### Configuration
- API base URL via `API_URL` env var (default `http://localhost:3001`)
- Process one event at a time (sequential, not parallel)

## Acceptance Criteria
- [ ] Agent runner starts and polls for pending wake events
- [ ] Creating a wake event → runner picks it up and marks it processing
- [ ] Runner spawns `claude` CLI with correct flags
- [ ] Text output from Claude is posted as comments on the task
- [ ] Actor's `session_id` and `last_token_count` are updated after run
- [ ] Events are marked `done` on success, `failed` on error
- [ ] Skips non-CTBACEO agents when project is in `analysing`/`paused`
