# Task 10 — Tasks CRUD Module

## Type: Backend

## Description
Build the Tasks NestJS module with CRUD, status management, assignment, and subtask support.

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/features/:featureId/tasks` | List tasks for a feature |
| GET | `/tasks/:id` | Get task with subtasks |
| POST | `/features/:featureId/tasks` | Create task |
| POST | `/tasks/:parentId/subtasks` | Create subtask |
| PATCH | `/tasks/:id` | Update task (status, assignment, etc.) |
| DELETE | `/tasks/:id` | Delete task |
| GET | `/projects/:projectId/tasks` | List all tasks for a project (across hierarchy) |

## Status Transitions
```
backlog → assigned
assigned → in_progress
in_progress → blocked | review_request | done
blocked → in_progress
review_request → in_progress | done
```

## Notes
- `PATCH /tasks/:id` with `{ assigned_agent_id }` sets status to `assigned` automatically
- `GET /tasks/:id` includes `subtasks` array and `assigned_agent` relation
- `GET /projects/:projectId/tasks` joins through feature → epic → sprint → mvp → project
- `parent_task_id` is nullable — top-level tasks have null, subtasks point to parent
- `created_by` is set from request body (actor ID)

## Acceptance Criteria
- [ ] Create task under a feature
- [ ] Create subtask under a task
- [ ] Status transitions enforced — invalid transitions return 400
- [ ] Assigning agent auto-sets status to `assigned`
- [ ] `GET /projects/:projectId/tasks` returns all tasks across the hierarchy
- [ ] `GET /tasks/:id` includes subtasks and assigned agent
